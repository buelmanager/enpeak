# Project Context 작성 규칙

이 문서는 project_context.md에 어떤 내용이 들어갈 수 있는지 판단 기준을 정의한다.

---

## 포함 기준 (이 프로젝트에 해당하면 포함)

1. **이 프로젝트에만 적용되는 설정/규칙**
2. **기술 스택 관련 정보** (언어, 프레임워크, 라이브러리)
3. **프로젝트 폴더 구조**
4. **코딩 컨벤션** (이 프로젝트만의 스타일)
5. **환경 설정** (개발/스테이징/프로덕션)
6. **주요 기능 및 도메인 지식**

---

## 제외 기준 (root_context로 가야 함)

1. 모든 프로젝트에 공통 적용되는 보편적 원칙
2. 언어/프레임워크에 무관한 일반 규칙
3. Claude와의 협업 방식 (전역 설정)

## 업데이트 절차

사용자가 새로운 내용을 project_context에 추가 요청 시:

1. **분류 확인**: root_context vs project_context 어디에 속하는지
2. **행동 분류**: 어떤 행동에 대한 룰인지 파악
3. **룰 파일 처리**:
   - 새로운 행동 -> `rules/[행동명].md` 생성
   - 기존 행동 -> 해당 룰 파일 업데이트
4. **참조 추가**: project_context.md의 "행동별 룰 참조" 테이블에 추가

---

## 룰 파일 작성 가이드

각 룰 파일(`rules/*.md`)은 다음 구조를 따른다:

```markdown
# [행동명] 룰

## 목적
이 룰이 왜 필요한지

## 규칙
1. 구체적인 규칙 1
2. 구체적인 규칙 2
...

## 예시
실제 적용 예시
```
